WF4 Publisher — Architecture

How all the pieces connect when you click Approve

Publishing flow

1

Admin clicks Approve

approveDraft() sets draft status → approved, then POSTs to WF4 webhook

2

n8n WF4 wakes up

Fetches draft + all 6 platform_posts from Supabase

3

Posts to all 6 platforms in sequence

Each node has Continue on Fail — one failure never blocks the others

→ Instagram → Facebook → WhatsApp → X → LinkedIn → Pinterest
4

WF4 updates Supabase

Each platform_post gets published or failed + posted_at timestamp

content_draft status → published

5

Dashboard updates in real-time

Supabase Realtime fires → card moves from Queue to Published tab with per-platform ✓/✗ badges

Code changes (minimal)

+

+
